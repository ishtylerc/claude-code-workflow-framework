# Templates

Blank and populated templates for the daily note system, plus a step-by-step walkthrough for prepending entries correctly.

## Files

| File | Purpose |
|------|---------|
| `daily-note-blank.md` | What a fresh daily note looks like right after the automation script creates it at midnight |
| `daily-note-populated-example.md` | A real daily note with tasks, notes entries, and ideas — shows what it looks like after a day of work |
| `weekly-note-blank.md` | Blank weekly summary template (auto-generated by the script) |
| `quarterly-note-blank.md` | Blank quarterly strategic plan template (auto-generated) |
| `annual-note-blank.md` | Blank annual strategic plan template (auto-generated) |
| `PREPEND-WALKTHROUGH.md` | **The important one** — step-by-step guide for Claude on how to prepend new entries to today's Notes section |

## Why This Directory Exists

The `skills/daily-note-management/` directory has the protocol and 10 real-world examples. This `templates/` directory complements that with:

1. **Blank templates** — so you can see exactly what a fresh note looks like before any entries are added
2. **A populated example** — so you can see what the same note looks like after a day of work
3. **A prepend walkthrough** — the precise Edit tool pattern for adding entries to the top (not bottom) of the Notes section

## How Claude Uses These

When the daily-note-management skill fires, Claude:
1. Reads `SKILL.md` for the 7-step protocol
2. References `entry-format.md` for the entry template
3. Uses `PREPEND-WALKTHROUGH.md` for the exact Edit tool pattern if it needs a refresher
4. Can look at `daily-note-populated-example.md` to see what a real note looks like in context

## How Humans Use These

- **Setting up a new vault?** Copy `daily-note-blank.md` into your vault structure as `YYYY/QX/[W]##/MM-DD-YYYY.md` to bootstrap manually
- **Want to see what it looks like populated?** Read `daily-note-populated-example.md`
- **Debugging why Claude appends instead of prepends?** Point Claude at `PREPEND-WALKTHROUGH.md`
- **Adapting to your needs?** Edit these templates to match your preferred sections, then update `scripts/daily-note-automation-v2.sh` to match
